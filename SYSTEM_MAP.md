# Magelo repository — system map

**Purpose:** Single architectural context for AI and humans. Logic and data contracts, not code dumps.  
**Canonical runbook:** `.github/workflows/daily-update.yml` (workflow name: **Daily Spell Inventory Update**).

---

## 1. End-to-end data flow

### 1.1 Upstream source

- **TAKP Magelo export** (`https://www.takproject.net/magelo/export/`): scheduled server-side dump; CI scrapes the page for the “last updated” timestamp to drive caching and labels.
- **Tab-separated text files** (UTF-8, often with BOM — pipelines use `utf-8-sig` where needed):
  - `character/TAKP_character.txt` — one row per character; header row defines column names.
  - `inventory/TAKP_character_inventory.txt` — rows keyed by character `id`, with `slot_id`, `item_id`, `item_name`.

### 1.2 Local / repo inputs (not from the live export)

- **`spell_exchange_list.json`** — PoK turn-in spell catalog (also searched under `quests/poknowledge/`). Required for spell inventory generation.
- **`spell_focii_level65.json`** — spell focus definitions (items, categories, percentages). CI copies from adjacent `Server/` checkout when present; required for class rankings focus scoring.
- **`data/item_stats.json`** — item id → stats, names, class flags, optional `focusSpellName` / `focusPct` / `focusPercentage` for ornate/elemental merges into focus lookup.
- **`data/item_name_to_id.json`** — normalized name → id (from `merge_dkp_loot_into_raid_sources.py --write-name-to-id`); backfills missing `item_id` in inventory and focus metadata.
- **`data/weapon_ranking_presets.json`**, **`data/dps_config.json`**, **`data/weapon_procs.json`**, **`data/weapon_threat_server.json`** — consumed by `scripts/ranking_weapon_engine.py` for WeaponMetric (melee/hybrid weapon scoring).
- **Optional:** `data/elemental_slot_options.json`, `data/dkp_prices.json`, `data/elemental_display_names.json`, DKP elemental mapping files for display and DKP overlays.

### 1.3 Two parallel consumers of the same dump

| Path | Script | Role |
|------|--------|------|
| **Spell inventory + deltas + leaderboards** | `generate_spell_page.py` | Mule/officer spell coverage, serverwide **delta** HTML, mob tracker, weekly/monthly leaderboards, `delta_snapshots/` baselines and daily gzipped JSON. Uses `parse_character_data` / `parse_inventory_file` (tab-split, fixed column indices for delta subset). |
| **Class rankings** | `generate_class_rankings.py` | Level **65** characters only; reads the same `character/` + `inventory/` files with **CSV DictReader** on column **names**; outputs **`class_rankings.json`** only from this script. |

### 1.4 Generated artifacts (high level)

- **`spell_inventory.html`** → deployed as site **`index.html`**.
- **`delta.html`**, **`delta-history.html`**, **`mob_tracker.html`**, **`leaderboard_*.html`**, **`dps_calculator.html`**, etc., as produced by `generate_spell_page.py` and prep steps.
- **`class_rankings.json`** — produced by `generate_class_rankings.py`; embeds per-character scores, focii, inventory, `class_weights`, `normalized_class_weights`, `focus_candidates`.
- **`class_rankings.html`** — **not** generated by Python in this repo; it is **gitignored** and is the static shell that loads `class_rankings.json` and client assets. CI copies it to `deploy/` **if the file exists** in the workspace.
- **`delta_snapshots/`** — `delta_daily_YYYY-MM-DD.json.gz`, `baseline_master*.json.gz`, weekly/monthly baseline JSON; some files are **committed** by the workflow when new dailies appear.

## 2. CI: trigger → build → deploy

**Workflow file:** `.github/workflows/daily-update.yml`  
**Jobs:** `update-spell-inventory` (build) → `deploy` (GitHub Pages).

### 2.1 Triggers

- **Schedule:** `cron: 40 16 * * *` (UTC), aligned after upstream Magelo refresh.
- **`workflow_dispatch`** — manual.
- **`push` to `main`** — full pipeline runs (ensures deploy path stays green).

### 2.2 Build job (conceptual order)

1. **Checkout**; **Python 3.11**.
2. **Export page probe** — curl Magelo export page; parse “updated” time; normalize to `YYYY-MM-DD` for cache keys and labels.
3. **Cache choreography** — GitHub Actions `actions/cache` on paths under `character/`, `inventory/`, `.magelo_update_date`:
   - Restore **yesterday’s** dump → copy to `TAKP_character_previous.txt` / `TAKP_character_inventory_previous.txt` **before** today’s files overwrite them (enables real diffs).
   - Restore or **download** today’s `TAKP_character.txt` and `TAKP_character_inventory.txt` from fixed export URLs when the cache key says data is stale.
   - **Backup today’s download** when needed so historical backfill steps cannot replace “today” with older cache restores.
4. **Historical backfill (optional)** — restores pinned dates (e.g. 2026-02-07, 2026-02-08) from cache to generate missing `delta_daily_*.json.gz` without polluting the main run; **restore today’s dump again** before the main generator.
5. **Baseline cache** — restore `delta_snapshots/baseline_master*.json.gz` and weekly/monthly baselines (used for leaderboards and date-range logic).
6. **`generate_spell_page.py`** — env `MAGELO_UPDATE_DATE` set from the export page. Produces spell inventory, delta HTML, snapshots, trackers, etc.
7. **Class rankings** — `merge_dkp_loot_into_raid_sources.py --write-name-to-id` (best-effort); then `generate_class_rankings.py` if `spell_focii_level65.json` is available.
8. **Commit** — new `delta_snapshots/delta_daily_*.json.gz` may be committed by `github-actions[bot]` (small JSON history in repo).
9. **Regenerate `data/item_name_to_id.json`** again for deploy consistency.
10. **Prepare `deploy/`** — copy `spell_inventory.html` → `deploy/index.html`, copy other HTML, `class_rankings.*`, `data/*.json` subsets, `delta_snapshots/*.json.gz`, `raid_item_sources.json`, `mob_tracker_deaths.json`, etc.; `touch deploy/.nojekyll`.
11. **Upload Pages artifact** — `actions/upload-pages-artifact` (main branch only; short retention).

### 2.3 Deploy job

- Runs only on **`main`**.
- **`actions/deploy-pages`** publishes the artifact to **GitHub Pages** (environment `github-pages`).
- Live URL pattern: `https://<owner>.github.io/<repo>/` (class rankings: `/class_rankings.html`).

### 2.4 Permissions / infra notes

- **`steps.date-info`** (week/month boundaries) is defined **late** in the job; an **earlier** “Restore baseline cache” step still references `date-info.outputs.month_start` for the cache key. In practice the empty first segment falls through to **`restore-keys: magelo-baseline-`** until the later “Cache baselines” step runs with a resolved month. If baseline caching misbehaves, compare both cache steps in `daily-update.yml`.

- **`contents: write`** — bot commits delta JSONs.
- **`pages: write`** + **`id-token: write`** — OIDC for Pages deployment.
- **Concurrency** — `update-spell-inventory` group cancels in-progress runs to avoid overlapping deploys.
- **Mob tracker persistence** — workflow may **curl** existing `mob_tracker_deaths.json` from the live site before merge.

## 3. Character / inventory handling (by component)

### 3.1 `generate_class_rankings.py`

- Filters **`level == 65`**; builds `characters[char_id]` with **`stats`** (see §4).
- Inventory: all rows for known characters; includes **bags** (important for clickies and pet power).
- Joins focus data: `spell_focii_level65.json` + **`ITEM_FOCUS_OVERRIDES`** + merges from **`data/item_stats.json`** (`focusSpellName` / focus pct).
- **WeaponMetric:** imports `ranking_weapon_engine.compute_weapon_ranking_metrics` with item stats and threat/DPS JSONs; Warrior score blends class-relative **hate/sec** and **DPS** (80/20); other weapon classes use normalized **`focus_raw_buffed`**.
- **Output:** `class_rankings.json`; may refresh `data/elemental_display_names.json` via `_write_elemental_display_names`.

### 3.2 `generate_spell_page.py`

- **Mules:** restricted character name lists for PoK spell inventory.
- **Deltas:** serverwide `parse_character_data` for **previous vs current** files; builds HTML + **`delta_storage`** persistence (daily gz, master baseline, week/month baselines).
- **Inventory parser:** `char_id` → list of `{slot_id, item_id, item_name}` keyed by name map from character file.

### 3.3 `delta_storage.py`

- Baseline and delta **JSON** formats (weekly/monthly/daily), gzip for dailies, helpers for date-range and leaderboard comparisons. Not the same code path as class rankings scoring.

## 4. Stat schema (class rankings / export)

### 4.1 Primary stats (classic EverQuest attributes)

Read from the TAKP character row as **total** columns (with `_row_get` for header variants):

| Field | Source column(s) | Notes |
|--------|-------------------|--------|
| **STR** | `STR_total` | |
| **STA** | `STA_total` | |
| **AGI** | `AGI_total` | |
| **DEX** | `DEX_total` | |
| **WIS** | `WIS_total` | |
| **INT** | `INT_total` | |
| **CHA** | `CHA_total` | |

Stored under each character’s **`stats`** object in `class_rankings.json`.

### 4.2 Secondary / combat-related stats (aggregated for scoring & UI)

| Concept | Source / encoding | Use |
|--------|-------------------|-----|
| **HP** | `hp_max_total` | Raw; class-max normalized to **`hp_pct`** in scoring. |
| **Mana** | `mana_max_total` | Classes with mana only; **`mana_pct`**. |
| **AC** | `ac_total` | **`ac_pct`** where class weights include AC. |
| **ATK** | `atk_item` + `atk_item_cap` → string `"current / cap"` | Melee/hybrid: **`atk_pct`** = min(current/250×100, 100) when cap is 250-style; also folded into **WeaponMetric** paths. |
| **Haste (item)** | `haste_item` | Binary **`haste_pct`**: 100% if ≥ 30 (item haste contributing to haste cap), else 0; raw in **`haste_value`**. |
| **FT (Flowing Thought)** | `mana_regen_item`, `mana_regen_item_cap` | String `"current / cap"` in **`mana_regen_item`**; **`ft_pct`**, **`ft_capped`**; focus **`FT`** when mana class. |
| **Resists (total)** | `MR_total` + `FR_total` + `CR_total` + `DR_total` + `PR_total` | Single sum **`resists`**; **`individual_resists`** keeps per-resist breakdown. |

### 4.3 Focus / “tertiary” scoring (not raw stats)

- Derived from **inventory × focus tables**: categories like Spell Damage, Spell Mana Efficiency, Spell Haste (subcategories: Bene, Det, All, Affliction, Enhancement), durations, range, pet power, bard instruments, Warrior Raex chest, Paladin/SK Shield of Strife, etc.
- **`CLASS_WEIGHTS`** in `generate_class_rankings.py` defines relative weights; **`normalize_class_weights`** scales **stats to ~65%** and **focus to ~35%** of the overall bar (Bard can override focus share via **`target_focus`**).
- **`focus_scores`**, **`focus_details`**, **`focus_total_points`**, **`focus_overall_pct`** (class-relative after post-pass) live in **`scores`**.

## 5. `class_rankings.json` top-level shape

- **`characters`**: array of `{ id, name, guild, class, race, stats, individual_resists, focii, damage_focii, mana_efficiency_cats, spell_haste_cats, duration_cats, scores, overall_score, inventory, class_rank, overall_rank, … }`.
- **`class_weights`**: raw **`CLASS_WEIGHTS`** from Python.
- **`normalized_class_weights`**: per-class normalized weights for UI (stat vs focus split).
- **`focus_candidates`**: items that could supply missing focuses (for tooltips / upgrade paths).
